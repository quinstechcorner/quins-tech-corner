Fix random CIV 5 crashes on Linux
(>8 CPU threads).


Here's a really simple fix to try if you've been experiencing random crashes in CIV 5 on Linux, where it otherwise runs OK.

A lot of random crashes in CIV under Linux are caused by the ridiculous hard-coded 8 thread CPU limit imposed in the config file.

If your CPU has more than 8 threads, this fix should be all you need.


1. Open terminal & go to the CIV folder

If you've installed via Steam, the following command should open the folder we need.

cd ~/.local/share/Aspyr/Sid\ Meier\'s\ Civilization\ 5/

In this folder, we're looking for the file called config.ini.
If you want to check if it's there, you can do the ls command.


2. Edit config.ini

Use a terminal-based text editor to edit the config file.
Almost all Linux distros have the nano text editor, so let's do that.

nano config.ini

Once you're editing config.ini, find the part that saysMaxSimultaneousThreads.

Change the number to how ever many threads your CPU has. In my case it was 16 threads (8 cores)
If you're unsure, google your CPU - the manufacturer website should list how many threads it has.

To save the file, press CTRL + O and then enter.
Then exit the editor by pressing CTRL + X .


3. Test it

Doing this should fix your random crashes, if they were being caused by the 8 thread limit in the config file.

Hopefully it works, but if not, try the usual fixes.
Reinstalling, checking drivers, and checking game-file integrity via Steam.
